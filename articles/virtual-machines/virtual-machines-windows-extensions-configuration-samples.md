# <a name="azure-windows-vm-extension-configuration-samples"></a>Exemples de Configuration de Windows Azure machine virtuelle Extension

> [AZURE.SELECTOR]
- [PowerShell - modèle](virtual-machines-windows-extensions-configuration-samples.md)
- [Infrastructure du langage commun - modèle](virtual-machines-linux-extensions-configuration-samples.md)

<br>

Cet article fournit un exemple de configuration pour la configuration des Azure machine virtuelle Extensions pour Windows machines virtuelles.

Pour en savoir plus sur ces extensions, voir [vue d’ensemble des Extensions Azure machine virtuelle.](virtual-machines-windows-extensions-features.md)

Pour plus d’informations sur la création de modèles d’extension, voir [de création de modèles de l’Extension.](virtual-machines-windows-extensions-authoring-templates.md)

Cet article répertorie les valeurs de configuration attendu de certaines des Extensions de Windows.

## <a name="sample-template-snippet-for-vm-extensions-with-iaas-vms"></a>Extrait de modèle d’exemple pour Extensions machine virtuelle avec machines virtuelles IaaS.
L’extrait de modèle pour le déploiement des extensions se présente comme suit :

      {
      "type": "Microsoft.Compute/virtualMachines/extensions",
      "name": "MyExtension",
      "apiVersion": "2015-05-01-preview",
      "location": "[parameters('location')]",
      "dependsOn": ["[concat('Microsoft.Compute/virtualMachines/',parameters('vmName'))]"],
      "properties":
      {
      "publisher": "Publisher Namespace",
      "type": "extension Name",
      "typeHandlerVersion": "extension version",
      "autoUpgradeMinorVersion":true,
      "settings": {
      // Extension specific configuration goes in here.
      }
      }
      }

## <a name="sample-template-snippet-for-vm-extensions-with-vm-scale-sets"></a>Extrait de code exemple modèle pour les Extensions de machine virtuelle avec des jeux d’échelle machine virtuelle.

    {
     "type":"Microsoft.Compute/virtualMachineScaleSets",
    ....
           "extensionProfile":{
           "extensions":[
             {
               "name":"extension Name",
               "properties":{
                 "publisher":"Publisher Namespace",
                 "type":"extension Name",
                 "typeHandlerVersion":"extension version",
                 "autoUpgradeMinorVersion":true,
                 "settings":{

                 }
               }
              }
            }
          }

Avant de déployer l’extension Vérifiez la dernière version d’extension et remplacez la « typeHandlerVersion » par la version la plus récente en cours.

Reste de cet article fournit des exemples de configuration pour les Extensions de machine virtuelle Windows.

Avant de déployer l’extension Vérifiez la dernière version d’extension et remplacez la « typeHandlerVersion » par la version la plus récente en cours.

### <a name="customscript-extension-14"></a>Extension CustomScript 1.4.
      {
          "publisher": "Microsoft.Compute",
          "type": "CustomScriptExtension",
          "typeHandlerVersion": "1.4",
          "settings": {
              "fileUris": [
                  "http: //Yourstorageaccount.blob.core.windows.net/customscriptfiles/start.ps1"
              ],
              "commandToExecute": "powershell.exe -ExecutionPolicy Unrestricted -start.ps1"
          },
          "protectedSettings": {
            "storageAccountName": "yourStorageAccountName",
            "storageAccountKey": "yourStorageAccountKey"
          }
      }

#### <a name="parameter-description"></a>Description du paramètre :

- fileUris : liste des URL des fichiers qui sont téléchargés sur l’ordinateur virtuel par l’Extension séparés par une virgule. Aucun fichiers ne sont téléchargés si rien n’est spécifié. Si les fichiers se trouvent dans le stockage Azure, le fileURLs peuvent être marqués comme privés et la storageAccountName cliquant et storageAccountKey peuvent être passés en tant que paramètres privés pour accéder à ces fichiers.
- commandToExecute : [paramètre obligatoire] : il s’agit de la commande qui sera exécutée par l’Extension.
- storageAccountName : [paramètre facultatif] : nom de compte de stockage pour accéder à la fileURLs, s’ils sont marqués comme privés.
- storageAccountKey : [paramètre facultatif] : clé de compte de stockage pour accéder à la fileURLs, s’ils sont marqués comme privés.

### <a name="customscript-extension-17"></a>Extension CustomScript 1.7.

Reportez-vous à CustomScript version 1.4 pour description du paramètre. Version 1.7 prend en charge pour l’envoi de script parameters(commandToExecute) en tant que protectedSettings, auquel cas ils seront chiffrées avant l’envoi. paramètre « commandToExecute » peut être spécifié dans les paramètres ou protectedSettings mais pas dans les deux.

        {
            "publisher": "Microsoft.Compute",
            "type": "CustomScriptExtension",
            "typeHandlerVersion": "1.7",
            "settings": {
                "fileUris": [
                    "http: //Yourstorageaccount.blob.core.windows.net/customscriptfiles/start.ps1"
                ],
                "commandToExecute": "powershell.exe -ExecutionPolicy Unrestricted -start.ps1"
            },
            "protectedSettings": {
              "commandToExecute": "powershell.exe -ExecutionPolicy Unrestricted -start.ps1",
              "storageAccountName": "yourStorageAccountName",
              "storageAccountKey": "yourStorageAccountKey"
            }
        }

                      "UserName": "UsernameValue2",
                      "Password": "PrivateSettingsRef:Key2"
                  }
              }

                  "Key1": "PasswordValue1",
                  "Key2": "PasswordValue2"

          }
      }

### <a name="azure-diagnostics"></a>Diagnostics de Windows Azure

Pour plus d’informations sur la façon de configurer des diagnostics, voir [Azure Diagnostics Extension](virtual-machines-windows-extensions-diagnostics-template.md)

          {
            "publisher": "Microsoft.Azure.Diagnostics",
            "type": "IaaSDiagnostics",
            "typeHandlerVersion": "1.5",
            "autoUpgradeMinorVersion": true,
            "settings": {
              "xmlCfg": "[base64(variables('wadcfgx'))]",
              "storageAccount": "[parameters('diagnosticsStorageAccount')]"
            },
            "protectedSettings": {
            "storageAccountName": "[parameters('diagnosticsStorageAccount')]",
            "storageAccountKey": "[listkeys(variables('accountid'), '2015-05-01-preview').key1]",
            "storageAccountEndPoint": "https://core.windows.net"
          }
          }

Dans l’exemple ci-dessus, remplacez le numéro de version avec le numéro de version plus récente.
